Update and Change Policy

03/01/2025

Closure Next is committed to keeping its users informed about changes and updates to the project. Transparency is a core value, and we ensure that all modifications are communicated promptly to the community. Here’s how updates and changes are managed:

Changelog Publication:

Each modification made to the Closure Next repository triggers the creation of a changelog, which is then immediately published to the Closure Next Telegram channel. This ensures that all users and contributors are quickly informed about the latest changes, bug fixes, improvements, or new features.

Automated Changelog Process:

- Every time a modification is made to the repository (such as code updates, bug fixes, or new features), a cURL request is automatically sent to the Closure Next bot repository via GitHub Actions.

- The bot is then activated by these requests, and it generates and publishes the changelog message to the Closure Next Telegram channel.

This process is fully automated, and as soon as the changes are pushed to the Closure Next repository, the community is instantly notified via the Telegram channel, keeping everyone updated in real-time.

Benefits of This System:

- Real-Time Updates: All changes are communicated promptly to the community, without any delays.

- Transparency: Every modification is made public and accessible, ensuring that users and contributors can follow along with the project's progress.

- Automation: The automated system ensures efficiency and consistency in the publication of changelogs.

For any questions or further details about changes, you can always refer to the changelog in the Telegram channel, where each update is detailed.
